# Fanqie Hot Novel Benchmark Notes

## Scope And Copyright Boundary

This note is a structural benchmark for the `episode-plan-lab` framework. I did
not download or store full copyrighted chapters. The analysis uses:

- public ranking pages and official synopses
- public chapter directories / chapter titles
- small public preview snippets only as orientation

The goal is not to reproduce text. The goal is to learn how hot serialized
openings turn a premise into a 10-20 chapter event container.

## Benchmark Observation

The strongest common pattern is not "one beat becomes one chapter." It is:

```text
high-concept premise
-> external event container
-> repeated tests / visible validations
-> resource or status gain
-> social confrontation
-> wider-world gate
```

The early 10-20 chapters are a compact arena. They make the reader understand
the rules by watching the protagonist repeatedly collide with a rule, not by
reading explanations.

## Case 1: `大一实习，你跑去749收容怪物`

### Public Signals

The 2026 ranking page places it in the top list and describes a college
internship premise where a construction site uncovers a bleeding stone, and the
protagonist directly approaches it instead of running. The public directory
shows the first 20 chapters moving through:

- long-awaited golden finger
- first containment / first reward
- 749 organization arrival
- joining / assessment
- assessment arena dominance
- first formal mission
- escalating containment incidents

### Framework Reading

`episode_outline`

- Container: "internship/749 onboarding and first monster containment."
- Public rule/deadline: prove whether the protagonist can enter the official
  abnormal-containment system.
- Reader promise: monster encounter produces immediate power/resource reward.

`character_threads`

- Protagonist line: bored hidden-power holder -> finally has usable target ->
  official investigator candidate.
- Institution line: 749 tests whether this dangerous outsider can be absorbed.
- Rival/peer line: ordinary candidates or staff become comparison objects.

`world_rule_tracks`

- Monsters can be contained or killed.
- Containment produces usable power / method / status.
- Official bureaucracy can legitimize violence if the protagonist passes its
  gates.

`event_pool`

- construction-site anomaly
- first kill/containment
- official visit
- benefits and job offer
- exam/assessment
- arena dominance
- first mission dispatch
- enemy/monster escalation

`episode_chapter_plan_set`

- The chapter set is not a slice of a beat. It is an onboarding tournament:
  each chapter either validates power, increases status, unlocks rules, or
  gives the next target.

### Lesson For Our Framework

The event container must include a status gate. "Can he join 749?" is more
powerful than "he encounters monsters." The chapter plan set should show
institutional validation, not just action scenes.

## Case 2: `时停起手，邪神也得给我跪下！`

### Public Signals

The 2026 ranking page summarizes a future wasteland premise: the protagonist
has one minute of time stop per day, which can accumulate. The public directory
shows early chapters centered on time-stop limits, fate/game-like tests,
bullets, combat risk, becoming extraordinary, town-destruction crisis, and the
limits of the ability.

### Framework Reading

`episode_outline`

- Container: "wasteland survival and first public proof of impossible time
  power."
- Public rule/deadline: one minute of stopped time is both advantage and
  constraint.
- Reader promise: clever ability use creates impossible reversals.

`character_threads`

- Protagonist line: powerless-looking survivor -> risk gambler -> recognized
  extraordinary actor.
- Mentor/observer line: someone tests whether his madness is strategy.
- Threat line: wasteland enemies force him to spend time-stop under pressure.

`world_rule_tracks`

- Time stop has a strict budget.
- Extraordinary status has social consequences.
- Wasteland power is measured publicly through combat and survival.

`event_pool`

- first ability demonstration
- roulette/bullet risk
- public combat gamble
- forced rescue or refusal
- ability-limit reveal
- town crisis
- social recognition after madness

`episode_chapter_plan_set`

- This is an ability-stress container. Every chapter asks: how does one minute
  solve a situation it should not be enough to solve?

### Lesson For Our Framework

World rules should be expressed as constraints with visible costs. "Can stop
time" is less compelling than "only one minute, and spending it wrong kills
you."

## Case 3: `游戏入侵 / 抢男女主机缘`

### Public Signals

The 2026 ranking page lists `游戏入侵：抢男女主机缘会上瘾诶`. A Fanqie official
same-theme page, `游戏入侵：开局疯抢男女主机缘`, shows a first volume titled
`十日内测`, with chapter titles that move from穿书, profession/sequence,
stealing protagonist opportunities, hunting, record creation, targeting other
characters, public conflict, and guild confrontation.

### Framework Reading

`episode_outline`

- Container: "limited-time game beta before broader game invasion."
- Public rule/deadline: ten-day beta window; opportunities are scarce and
  front-loaded.
- Reader promise: protagonist uses future/book knowledge to steal resources
  before canon owners arrive.

`character_threads`

- Protagonist line: cannon-fodder / villain identity -> opportunity predator.
- Canon protagonist line: deprived of expected resources, becomes pressure.
- Guild/social line: the protagonist's theft generates public reputation and
  retaliation.

`world_rule_tracks`

- Game resources have first-claim advantage.
- Hidden jobs/skills/items reshape future power hierarchy.
- The game world will later invade reality, so early gains compound.

`event_pool`

- awaken into book/game premise
- identify first opportunity
- steal job/resource
- hunt boss
- create record
- misdirect original protagonist
- steal second character's opportunity
- guild conflict
- public reputation tag
- revenge invitation

`episode_chapter_plan_set`

- This is a countdown resource-race. The plan group should track not just
  fights, but opportunity ownership shifting from canon characters to the new
  protagonist.

### Lesson For Our Framework

An event pool needs ownership transfer: who was supposed to get the resource,
who gets it now, and what retaliation this creates.

## Case 4: `照片是舍友网恋对象是财阀继承人`

### Public Signals

The 2026 ranking page describes a穿书甜宠 premise: the protagonist wakes as an
evil supporting woman who used a roommate's photo to scam a wealthy heir online;
the original fate is death, but the male lead may already know. The Fanqie
official first chapter establishes immediate meeting pressure, identity debt,
money debt, borrowed photo risk, and the choice to delay rather than confess
immediately.

### Framework Reading

`episode_outline`

- Container: "borrowed-photo online romance about to become offline meeting."
- Public rule/deadline: the online target wants to meet; the scam must survive
  contact with reality.
- Reader promise: danger and comedy come from every solution creating a bigger
  lie.

`character_threads`

- Protagonist line: survival-by-delay -> active deception manager -> reluctant
  emotional entanglement.
- Male lead line: apparent victim -> possible knowing participant -> controlled
  hunter.
- Roommate/original-photo line: identity ownership and exposure risk.

`world_rule_tracks`

- The book plot predicts the protagonist's death.
- Wealth/status imbalance makes confession costly.
- Online identity is a fragile stage prop that will collapse under offline
  contact.

`event_pool`

- wake in dormitory
- discover borrowed-photo scam
- receive meeting demand
- calculate debt/fate
- attempt delay
- roommate/photo risk
- first near-meeting
- money transfer raises temptation
- male lead's knowledge hint
- next lie becomes harder to sustain

`episode_chapter_plan_set`

- This is a deception-deadline container. The first 10-20 chapters should not
  only "flirt"; they should tighten the lie until a controlled reveal or
  counter-reveal becomes unavoidable.

### Lesson For Our Framework

For relationship/comedy openings, the equivalent of "battle reward" is not
always victory; it can be temporary survival, higher debt, stronger attraction,
and a more dangerous next lie.

## Case 5: `我不是戏神`

### Public Signals

The 2026 ranking page summarizes a red-meteor premise where modern civilization
collapses and a gray mirror-like world drags order into chaos. A public preview
shows an opening built around a striking visual body-state image: a red-robed
figure in storm/mud, amnesia, noise, and theatrical identity.

### Framework Reading

`episode_outline`

- Container: "identity awakening inside a collapsed / theatrical apocalypse."
- Public rule/deadline: the protagonist must understand who he is before the
  surrounding world assigns him a dangerous role.
- Reader promise: spectacle, mystery, and identity power are fused.

`character_threads`

- Protagonist line: amnesiac body -> role recognition -> agency through
  performance.
- World-threat line: disaster is not background; it presses the protagonist into
  a role.
- Witness/opponent line: others respond to the protagonist as omen, monster, or
  savior.

`world_rule_tracks`

- Civilization collapse has a metaphysical rule.
- Theater/performance is not decoration; it is a power grammar.
- Identity naming changes what the protagonist can do.

`event_pool`

- sensory awakening
- identity question
- hostile environment
- first supernatural reaction
- public misrecognition
- role/power hint
- first choice to perform or resist role
- wider apocalypse signal

`episode_chapter_plan_set`

- This is not an onboarding exam but an identity-stage container. The first
  chapters must keep asking: "what role is he being forced to play, and what
  happens when he performs it?"

### Lesson For Our Framework

For spectacle/mystery works, the external event container can be an atmosphere
with rules, but it still needs concrete stage events. Vibe alone is not enough.

## Cross-Case Pattern

The useful benchmark rule for our architecture:

```text
episode_outline must name the arena:
  exam / onboarding / beta countdown / deception deadline / identity stage

character_threads must be roles inside that arena:
  candidate / predator / witness / target / owner / examiner / rival

world_rule_tracks must create constraints:
  one-minute limit / first-claim ownership / public anchor / official validation

event_pool must be writable tests:
  not "relationship develops", but "photo demand arrives", "ring returns",
  "assessment starts", "first boss dies", "paper proof fades"

episode_chapter_plan_set must show cadence:
  hook -> test -> visible result -> reward/cost -> next gate
```

## What This Means For NovelAI

Our current 5-product schema is basically right, but the prompt needs to keep
forcing:

1. A named external arena, not just a situation.
2. A visible rule/deadline by chapter 1-2.
3. Repeated test events, not premise explanation.
4. Ownership transfer for resources/status/identity.
5. Social validation or public pressure, because hot openings often make power
   legible through other people's reactions.
6. A larger-world gate by the end of the plan set.

I added this benchmark lesson into `src/prompts/episode-plan-lab.ts`:

```text
Opening-episode benchmark lesson from hot serialized webnovels:
the first 10-20 chapters should usually form a named container with a public
rule/deadline, fast resource or ability validation, repeated visible
wins/setbacks, social pressure, and a gate into the larger world.
```
